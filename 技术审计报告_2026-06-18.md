### Prisma Client 连接方式

```typescript
// lib/prisma.ts — 全局单例模式
const globalForPrisma = globalThis as unknown as {
  prisma: PrismaClient | undefined;
};
export const prisma = globalForPrisma.prisma ?? new PrismaClient();
if (process.env.NODE_ENV !== "production") globalForPrisma.prisma = prisma;
```

- 开发环境：PrismaClient 挂载到 `globalThis`，避免 Next.js HMR 热重载导致重复实例
- 生产环境：每次请求创建新实例（Vercel Serverless 无状态特性）

## 5. Prisma 实际 Schema 使用情况

### 当前生效的 schema 文件

| 环境 | 生效文件 | Provider |
|------|----------|----------|
| **本地 dev** | `prisma/schema.prisma` | `sqlite` |
| **Vercel build** | `prisma/schema.prisma`（构建时由 `schema.pg.prisma` 覆盖） | `postgresql` |

### 两个 Schema 文件的差异

| 文件 | Provider | 行数 | 用途 |
|------|----------|------|------|
| `schema.prisma` | `sqlite` | 307 | 本地开发主文件 |
| `schema.pg.prisma` | `postgresql` | 307 | Vercel 生产构建用 |
| `schema.sqlite.prisma.bak` | — | 307 | 临时备份（已被删除） |

两个文件结构完全一致（模型定义、枚举、关系均相同），唯一区别是 datasource provider。

### 数据模型总览

**13 个模型：**

```
User                    — 用户认证与权限（NextAuth + 角色 + 头像 + 权限JSON）
Series                  — 七序产品体系
Works                   — 作品（关联 Series）
WorksAssets             — 作品资源（图片/文件等）
Products                — 商品（关联 Works）
ProductSku              — SKU（关联 Products）
ProductCost             — 产品成本核算
RawMaterial             — 原材料（关联 Bom、采购记录、库存流水）
PurchaseRecord          — 采购记录
InventoryTransaction    — 库存流水
Bom                     — BOM 物料清单
Customer                — 客户管理（多来源渠道）
Order                   — 订单管理（多渠道、状态流转、动态商品行）
```

**6 个枚举：**

```
OrderChannel        — MINIPROGRAM / STANDALONE / MANUAL（订单渠道）
OrderStatus         — PENDING / CONFIRMED / IN_PRODUCTION / SHIPPED / COMPLETED / CANCELLED（订单状态流转）
```
